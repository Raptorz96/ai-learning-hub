Implementazione con Make.com - Guida Completa

👋 Mai usato Make.com? Nessun problema! È come costruire con i LEGO - trascini moduli, li colleghi, e funziona! Ti guiderò passo passo.
1

🔑 Crea Account OpenWeatherMap

Prima di tutto ci serve una chiave API per accedere ai dati meteo reali:

  1. Vai su OpenWeatherMap
  2. Clicca su "Sign Up" (è gratis!)
  3. Conferma la tua email
  4. Vai su "API Keys" nel tuo profilo
  5. Copia la tua chiave API (sembra: abc123def456...)
⚠️ Importante: La chiave API può impiegare 10-15 minuti per attivarsi. Nel frattempo, continua con i prossimi step!
2

🚀 Setup Scenario Make.com

Ora creiamo il nostro "scenario" (workflow) in Make:

  1. Registrati su Make.com (piano gratuito)
  2. Clicca su "Create a new scenario"
  3. Dai un nome tipo "Assistente Meteo AI"
  4. Sei nell'editor visuale - qui costruiremo tutto!

3

📥 Configura Webhook Trigger

Il webhook è il punto di ingresso del nostro agente - è come dare un indirizzo web al tuo assistente:

  1. Clicca sul grande "+" al centro dello schermo
  2. Nella barra di ricerca scrivi "Webhooks" e clicca sull'icona
  3. Seleziona "Custom webhook"
  4. Clicca sul pulsante "Add"
  5. Dai un nome tipo "Weather Assistant Input"
  6. IMPORTANTE: Copia e salva l'URL che appare (es: https://hook.eu1.make.com/abc123...)
⚠️ Non chiudere ancora! Dobbiamo dire a Make che tipo di dati riceverà:
  1. Lascia la finestra del webhook aperta
  2. Apri un nuovo tab del browser e vai su httpie.io/app (versione web gratuita)
  3. Incolla il tuo webhook URL
  4. Cambia metodo da GET a POST
  5. Nel body, incolla: {"city": "Roma"}
  6. Clicca "Send"
  7. Torna su Make - vedrai "Successfully determined" ✅
  8. Ora clicca "Save" per chiudere

🎯 Cosa abbiamo fatto: Make ora sa che riceverà oggetti con il campo "city"!

4

🌍 Aggiungi la Chiamata API Meteo

Ora colleghiamo OpenWeatherMap per ottenere dati meteo reali:

  1. Clicca sul cerchietto a destra del modulo Webhook (apparirà una linea)
  2. Apparirà un nuovo "+" - cliccalo
  3. Cerca "HTTP" e seleziona l'icona blu
  4. Scegli "Make a request"
  5. Compila i campi come segue:

URL:

https://api.openweathermap.org/data/2.5/weather

Method: GET (già selezionato)

Query String (clicca "Add parameter" per ogni riga):

  • • Key: q → Value: clicca nel campo e seleziona city dal menu che appare
  • • Key: appid → Value: incolla la tua API key di OpenWeatherMap
  • • Key: units → Value: metric
  • • Key: lang → Value: it

Parse response: ✅ Yes (assicurati sia attivo!)

Clicca "Save" per salvare il modulo.

💡 Cosa fa questo modulo: Quando riceve una città, chiede a OpenWeatherMap il meteo attuale in italiano con temperature in Celsius.
5

🤖 Connetti OpenAI per le Risposte

Ora aggiungiamo l'intelligenza artificiale che trasformerà i dati meteo in risposte amichevoli:

  1. Clicca sul cerchietto a destra del modulo HTTP
  2. Clicca sul nuovo "+" che appare
  3. Cerca "OpenAI" e seleziona l'icona
  4. Scegli "Create a Chat Completion"
  5. Clicca "Add" accanto a Connection per collegare il tuo account OpenAI:
    • Dai un nome alla connessione (es: "Il mio OpenAI")
    • Incolla la tua API key di OpenAI (ottienila qui)
    • Clicca "Save"
  6. IMPORTANTE: Clicca su "Show advanced settings" in basso
💳 Nota su OpenAI: Se è la prima volta che usi OpenAI, dovrai aggiungere un minimo di credito ($5) al tuo account. Ogni richiesta costa circa $0.002 (0,2 centesimi).

Configura i messaggi (clicca "Add message" sotto Messages):

Messaggio 1 - System:

  • • Role: System
  • • Content: copia questo testo:
Sei un assistente meteorologico italiano super amichevole!

Quando ricevi dati meteo per una città:
1. Rispondi sempre in modo conversazionale e simpatico
2. Dai consigli pratici su cosa indossare o fare
3. Usa emoji per rendere la risposta più vivace 😊
4. Mantieni le risposte concise ma informative

Rispondi SEMPRE in italiano.

Messaggio 2 - User (clicca "Add message" di nuovo):

  • • Role: User
  • • Content: costruisci questo messaggio usando i campi dinamici:

Nel campo Content, scrivi questo testo e inserisci i dati cliccando sui campi dal pannello a destra:

Dammi il meteo attuale per {{2.name}} con questi dati:
- Temperatura: {{2.main.temp}}°C
- Condizioni: {{2.weather[0].description}}
- Temperatura percepita: {{2.main.feels_like}}°C
- Umidità: {{2.main.humidity}}%
- Vento: {{2.wind.speed}} m/s
⚠️ Come inserire i campi dinamici:
  1. Clicca nel campo "Text Content" dove vuoi inserire un dato
  2. Nel pannello a destra, clicca su "HTTP - Make a request" (modulo 2)
  3. Trova e clicca sui campi che ti servono (name, main.temp, ecc.)
  4. I campi verranno inseriti automaticamente nel formato {{2.campo}}

Altre impostazioni:

  • Model: gpt-3.5-turbo (più economico)
  • Max Tokens: 300
  • Temperature: 0.7

Clicca "Save" per salvare il modulo.

6

📤 Invia la Risposta Finale

Ultimo passaggio! Dobbiamo inviare la risposta dell'AI a chi ha fatto la richiesta:

  1. Clicca sul cerchietto a destra del modulo OpenAI
  2. Clicca sul "+" che appare
  3. Cerca "Webhooks" di nuovo
  4. Questa volta scegli "Webhook response"
  5. IMPORTANTE: Clicca su "Show advanced settings" in basso

Configura la risposta:

Status: 200 (lascia così)

Headers (clicca "Add parameter"):

  • • Key: Content-Type
  • • Value: application/json

Body: costruisci questo JSON cliccando sui campi:

{
  "response": {{3.choices[0].message.content}},
  "city": {{2.name}},
  "success": true
}

Clicca "Save" per salvare il modulo.

🎉 Quasi finito! Il tuo workflow è completo. Ora dobbiamo solo salvarlo e testarlo!
7

💾 Salva e Attiva lo Scenario

IMPORTANTE - Non dimenticare questi passaggi!

  1. In basso a sinistra, clicca sul pulsante "Save" (icona dischetto)
  2. In alto a sinistra, clicca sull'interruttore per attivare lo scenario (diventa viola)
  3. Sempre in alto, clicca su "Run once" per testare
⚠️ Errori comuni:
  • Se vedi pallini rossi = c'è un errore in quel modulo, cliccaci e controlla
  • Se non funziona = controlla che lo scenario sia attivo (interruttore viola)
  • Se l'API meteo dà errore = aspetta 10-15 min che la chiave si attivi
8

🧪 Testa il Tuo Assistente!

È il momento della verità! Testiamo se tutto funziona:

Metodo 1 - Test Veloce (Web):

  1. Vai su httpie.io/app
  2. Incolla il tuo webhook URL
  3. Metodo: POST
  4. Body: {"city": "Milano"}
  5. Clicca "Send"

Metodo 2 - Test Pro (se hai HTTPie installato):

http POST YOUR_WEBHOOK_URL city="Roma"

Risposta di esempio che dovresti ricevere:

{
  "response": "Ciao! 🌤️ A Roma oggi c'è un bel sole con 22°C! 
               La temperatura è perfetta per una passeggiata. 
               Ti consiglio una maglietta leggera e occhiali da sole! 😎",
  "city": "Roma",
  "success": true
}

Costi e Ottimizzazioni

💰 Stima Costi Mensili

  • Make.com Free: 1000 operazioni/mese
  • OpenWeatherMap Free: 1000 call/giorno
  • OpenAI: ~$0.002 per richiesta

Totale: ~$2/mese per 1000 richieste

💡 Consigli per Risparmiare

  • Usa sempre gpt-3.5-turbo (non gpt-4)
  • Limita Max Tokens a 200-300
  • Testa prima di lasciarlo sempre attivo
  • Disattiva lo scenario quando non serve

❓ Problemi Comuni

L'API meteo dà errore 401:

→ La chiave API non è ancora attiva. Aspetta 10-15 minuti.

OpenAI dà errore di billing:

→ Devi aggiungere credito al tuo account OpenAI (minimo $5).

Il webhook non riceve dati:

→ Assicurati che lo scenario sia attivo (interruttore viola).

La risposta è in inglese:

→ Controlla che nel system prompt ci sia "Rispondi SEMPRE in italiano".
