Sistemi Multi-Agent

Scopri come orchestrare team di agenti specializzati per creare sistemi AI complessi e scalabili. Manager pattern, decentralized pattern e best practices per la collaborazione.

Perché Multi-Agent?

Immagina di gestire un ristorante. Hai bisogno di un cuoco, un cameriere, un cassiere. Ognuno è specializzato nel suo lavoro. Lo stesso vale per i sistemi AI: invece di un singolo agente che fa tutto, puoi avere più agenti specializzati che collaborano.

🎯 Vantaggi dei Sistemi Multi-Agent

  • ✅ Specializzazione: Ogni agente eccelle nel suo dominio specifico
  • ✅ Scalabilità: Aggiungi nuovi agenti senza modificare quelli esistenti
  • ✅ Parallelismo: Più agenti possono lavorare simultaneamente
  • ✅ Manutenibilità: Aggiorna un agente senza toccare gli altri
  • ✅ Resilienza: Se un agente fallisce, gli altri continuano

🎯
Manager Pattern: Il Direttore d'Orchestra

Nel Manager Pattern, un agente centrale coordina tutti gli altri. Come un direttore d'orchestra che guida i musicisti, il manager sa quando chiamare ogni specialista.

Come Funziona il Manager Pattern

Utente
💬
"Traduci 'ciao' in spagnolo, francese e italiano"
↓
Manager Agent
→
🇪🇸 Agente Spagnolo
🇫🇷 Agente Francese
🇮🇹 Agente Italiano
Python 
from agents import Agent, Runner

# Definisci gli agenti specializzati
spanish_agent = Agent(
    name="spanish_translator",
    instructions="Sei un traduttore esperto di spagnolo. Traduci solo in spagnolo."
)

french_agent = Agent(
    name="french_translator", 
    instructions="Sei un traduttore esperto di francese. Traduci solo in francese."
)

italian_agent = Agent(
    name="italian_translator",
    instructions="Sei un traduttore esperto di italiano. Traduci solo in italiano."
)

# Crea il Manager che coordina tutti
manager_agent = Agent(
    name="manager_agent",
    instructions=(
        "Sei un agente di traduzione. Usi i tools disponibili per tradurre. "
        "Se ti vengono chieste più traduzioni, chiami i tool rilevanti."
    ),
    tools=[
        # Trasforma ogni agente in un tool per il manager
        spanish_agent.as_tool(
            tool_name="translate_to_spanish",
            tool_description="Traduci il messaggio dell'utente in spagnolo"
        ),
        french_agent.as_tool(
            tool_name="translate_to_french",
            tool_description="Traduci il messaggio dell'utente in francese"
        ),
        italian_agent.as_tool(
            tool_name="translate_to_italian",
            tool_description="Traduci il messaggio dell'utente in italiano"
        ),
    ],
)

# Esegui il sistema
async def main():
    msg = "Traduci 'ciao' in spagnolo, francese e italiano"
    result = await Runner.run(manager_agent, msg)
    print(f"Risultato: {result.new_messages[-1].content}")

# Output:
# Spagnolo: hola
# Francese: bonjour  
# Italiano: ciao

✨ Quando Usare il Manager Pattern

  • Vuoi un singolo punto di contatto con l'utente
  • Hai bisogno di coordinare risultati da più fonti
  • La logica di orchestrazione è complessa
  • Vuoi nascondere la complessità all'utente

Decentralized Pattern: La Staffetta

Nel Decentralized Pattern, gli agenti si passano il controllo come in una staffetta. Ogni agente sa quando è il suo turno e a chi passare il testimone.

Flusso del Decentralized Pattern

🎯

Triage Agent

Analizza la richiesta e decide chi deve gestirla

→
💼

Sales Agent

Gestisce ordini e informazioni prodotti

→
🔧

Support Agent

Risolve problemi tecnici e assistenza

→
📦

Order Agent

Traccia spedizioni e gestisce resi

Python 
from agents import Agent, Runner

# Agente Support
technical_support_agent = Agent(
    name="Technical Support Agent",
    instructions=(
        "Fornisci assistenza esperta per problemi tecnici, "
        "interruzioni di sistema o troubleshooting prodotti."
    ),
    tools=[search_knowledge_base]
)

# Agente Sales
sales_assistant_agent = Agent(
    name="Sales Assistant Agent",
    instructions=(
        "Aiuta i clienti a navigare il catalogo prodotti, "
        "raccomanda soluzioni adatte e facilita gli acquisti."
    ),
    tools=[initiate_purchase_order]
)

# Agente Orders
order_management_agent = Agent(
    name="Order Management Agent",
    instructions=(
        "Assisti i clienti con tracking ordini, "
        "date di consegna, resi e rimborsi."
    ),
    tools=[track_order, process_return]
)

# Agente Triage che smista le richieste
triage_agent = Agent(
    name="Triage Agent",
    instructions=(
        "Analizza la richiesta del cliente e determina quale agente "
        "specializzato può gestirla meglio. Passa il controllo all'agente appropriato."
    ),
    tools=[
        # Handoff tools per passare il controllo
        technical_support_agent.as_handoff_tool(
            tool_name="handoff_to_support",
            tool_description="Passa a supporto tecnico per problemi e troubleshooting"
        ),
        sales_assistant_agent.as_handoff_tool(
            tool_name="handoff_to_sales",
            tool_description="Passa a sales per informazioni prodotti e acquisti"
        ),
        order_management_agent.as_handoff_tool(
            tool_name="handoff_to_orders",
            tool_description="Passa a orders per tracking e gestione ordini"
        ),
    ],
)

# Esempio di conversazione
async def customer_service_flow():
    # L'utente inizia sempre con il Triage Agent
    msg = "Dov'è il mio ordine #12345?"
    
    # Il Triage Agent analizza e passa a Order Agent
    result = await Runner.run(triage_agent, msg)
    
    # Order Agent risponde direttamente all'utente
    print("Risposta:", result.new_messages[-1].content)
    # Output: "Il tuo ordine #12345 è in transito. Consegna prevista domani!"

🔄 Quando Usare il Decentralized Pattern

  • Workflow con passaggi sequenziali chiari
  • Ogni agente deve interagire direttamente con l'utente
  • Non serve una sintesi centralizzata
  • Vuoi maggiore autonomia per ogni agente

⚖️
Manager vs Decentralized: Quale Scegliere?

Caratteristica Manager Pattern Decentralized Pattern
Controllo Centralizzato nel manager Distribuito tra agenti
Comunicazione Manager ↔ Agenti Agente → Agente
Contesto Manager mantiene tutto Passato tra agenti
Scalabilità Limitata dal manager Molto scalabile
Complessità Semplice da capire Più complesso
Use Case Ideale Task paralleli, sintesi risultati Workflow sequenziali

💡
Best Practices per Sistemi Multi-Agent

🎯

Definisci Ruoli Chiari

Ogni agente deve avere uno scopo ben definito. Evita sovrapposizioni di responsabilità che possono creare confusione.

❌ "Gestisci richieste varie"
✅ "Gestisci solo resi e rimborsi ordini"
📝

Protocolli di Comunicazione

Standardizza come gli agenti si scambiano informazioni. Usa formati strutturati per evitare ambiguità.

{
  "from": "triage_agent",
  "to": "support_agent",
  "context": {...},
  "priority": "high"
}
🔍

Monitoring e Debug

Traccia ogni interazione tra agenti. È essenziale per capire cosa succede quando qualcosa va storto.

[TRIAGE] → Ricevuta richiesta ordine
[TRIAGE] → Handoff a ORDER_AGENT
[ORDER] → Query database ordini
[ORDER] → Risposta inviata

⚠️ Gestione Errori nei Multi-Agent

Con più agenti, gli errori possono propagarsi. Implementa sempre:

  • Fallback: Ogni agente deve avere un piano B
  • Timeout: Limiti di tempo per evitare attese infinite
  • Retry Logic: Riprova automaticamente in caso di errori temporanei
  • Graceful Degradation: Il sistema deve funzionare anche se un agente fallisce

🎯 Punti Chiave da Ricordare

1️⃣ Inizia semplice

Non partire subito con 10 agenti. Inizia con 2-3 e aggiungi gradualmente. La complessità cresce esponenzialmente con ogni agente.

2️⃣ Pattern != Soluzione

Manager e Decentralized sono punti di partenza. Puoi mescolarli: un manager che coordina sottogruppi decentralizzati, per esempio.

3️⃣ Osservabilità è cruciale

Con più agenti, il debugging diventa complesso. Investi in logging, tracing e monitoring fin dall'inizio. Ti ringrazierai dopo.
